# Glossary

## AI-assisted development
Using AI tools to support planning, coding, review, and documentation while keeping human ownership.

## Guardrails
Explicit instructions that constrain scope, allowed files, and output behavior.

## Scope drift
When work expands beyond the intended requirement or boundary.

## Evidence-first workflow
Validating changes through concrete outputs (tests, logs, status codes, commands) before acceptance.

## Break test
A deliberate code break used to prove tests fail correctly.

## Repo-grounded prompt
A prompt that references actual repository files and real context.

## Hallucinated API
AI-generated function/class/library behavior that does not exist in real code or docs.

## Governance prompt
A prompt designed to enforce review posture, constraints, and evidence requirements.

## Prompt quality
How clear, scoped, and verifiable a prompt is. Higher prompt quality generally improves output quality.

## Release readiness
Confidence that project artifacts (CI, Docker, docs, tests, handoff) are complete and trustworthy.

## Ownership statement
A written confirmation that the developer understands and accepts responsibility for final changes.

## Context engineering
Structuring and selecting the right context (files, constraints, examples) to improve AI reliability.
