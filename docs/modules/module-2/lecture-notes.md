## Key ideas
1. Use file-aware AI workflows in the IDE so prompts are grounded in actual project files.
2. Build backend in connected stages: model, storage, CRUD, business rules, tests.
3. Verify each stage before moving on to avoid compounding hidden errors.
4. Use break tests to prove tests are meaningful, not superficial.
