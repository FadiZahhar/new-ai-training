## Key ideas
1. Use file-aware AI workflows in the IDE so prompts are grounded in actual project files.
2. Build backend in connected stages: model, storage, CRUD, business rules, tests.
3. Verify each stage before moving on to avoid compounding hidden errors.
4. Use break tests to prove tests are meaningful, not superficial.

## Extracted context highlights from source
- Module flow is intentionally one continuous build with verification gates between each part.
- Prompt labels in the module are tied to staged workflow steps; keep prompt library and implementation steps aligned.
- The expected output is not only code: it includes proof artifacts for strict model behavior, endpoint correctness, business rules, and test quality.
