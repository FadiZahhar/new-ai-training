## Key ideas
1. AI coding assistants are useful for speed and recall, but they do not validate runtime correctness.
2. Common failure modes include hallucinated APIs, weak business logic, and missing non-functional requirements.
3. The developer keeps ownership: review, test, correct, and explain every accepted change.
4. Prompt quality directly impacts code quality; treat prompts like small technical specs.
