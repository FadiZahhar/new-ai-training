## Key ideas
1. Treat AI-generated infrastructure files as drafts until verified.
2. Follow strict review habits: inspect diffs, validate behavior, then accept.
3. Choose tools by scope (line-level, file-level, repo-level) intentionally.
4. Document technical decisions and tradeoffs in your own words.

## Extracted context highlights from source
- Module 4 expands scope from feature coding to repository operations: memory, CI, Docker, docs, and review triage.

- Expected deliverables include evidence chains (green -> red -> green CI, container health checks, and corrected documentation claims).
