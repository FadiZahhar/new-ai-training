# Final Project Requirement

## Source
- [Final Course Project Simplified (PDF)](../assets/pdf/final-course-project-simplified.pdf)

## Goal
Prepare the Task Tracker for teammate handoff and release-readiness with strong evidence of AI ownership.

## Core rules
1. Use the same repository from mid-course work.
2. Complete final work on branch `final-project`.
3. Do not add unrelated product features.
4. Keep ownership: if you cannot explain a change, do not submit it.

## Required repository elements
- `.github/workflows/ci.yml`
- `Dockerfile` and `.dockerignore`
- `README.md`
- `AGENTS.md`
- `app/`, `frontend/`, `tests/`
- `docs/release-evidence.md`
- `docs/final-ai-review.md`
- `docs/ai-playbook.md`

## Final evidence expectations
- Baseline backend/frontend/test checks
- Release-readiness evidence (CI + Docker + docs)
- Clear AI usage, review, and rejection/acceptance rationale
