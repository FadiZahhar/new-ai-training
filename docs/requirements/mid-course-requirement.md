## Goal
Extend the existing Task Tracker by adding **two scoped end-to-end features** while proving a disciplined AI-assisted engineering workflow.

## Mandatory constraints
1. Start from your existing Task Tracker repository.
2. Implement two small features end-to-end (at least one must be visible in the frontend).
3. Use full workflow per feature: plan, constrain, implement in steps, inspect AI output, run, test, and document decisions.
4. Prioritize completeness and explainability over feature ambition.

## Suggested feature options
- Due dates + overdue filter
- Tags / labels
- Search + combined filters
- Task comments
- Activity log
